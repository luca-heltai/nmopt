University of Pisa

Overview¶

This lecture is a numerical optimization toolbox for the reduced formulation

min⁡u∈Rnf(u).\min_{u\in\mathbb{R}^n} f(u).u∈Rnmin​f(u).
(1)

The same algorithms will later be reused for PDE-constrained optimization, where f(u)=J(S(u),u)f(u)=J(S(u),u)f(u)=J(S(u),u) and each gradient evaluation typically requires state/adjoint solves.

Main goals:

  1. put GD, nonlinear CG, BFGS, and trust-region in one common framework;

  2. isolate the role of Armijo/Wolfe conditions;

  3. compare methods by cost per iteration and robustness;

  4. prepare the transition to reduced PDE-constrained optimization.

Unified Iterative Template¶

Most unconstrained methods fit, at a high level, the same iterative template: Choose u0∈Rnu_0\in\mathbb{R}^nu0​∈Rn and for k=0,1,2,…k=0,1,2,\ldotsk=0,1,2,… compute

uk+1=uk+αkpk,u_{k+1}=u_k+\alpha_k p_k,uk+1​=uk​+αk​pk​,
(2)

with four design choices:

  • model of local “geometry”, i.e., what we know about the “landscape” of fff in the surrounding of uku_kuk​ (gradient only, Hessian, Hessian approximation);

  • search direction pkp_kpk​;

  • step acceptance/globalisation rule;

  • stopping criteria.

Typical stopping checks (often combined):

∥∇f(uk)∥≤εg,∣f(uk+1)−f(uk)∣max⁡(1,∣f(uk)∣)≤εf,∥uk+1−uk∥≤εu,k≤kmax⁡.\|\nabla f(u_k)\|\le \varepsilon_g,\qquad \frac{|f(u_{k+1})-f(u_k)|}{\max(1,|f(u_k)|)}\le \varepsilon_f,\qquad \|u_{k+1}-u_k\|\le \varepsilon_u, \qquad k\le k_{\max}.∥∇f(uk​)∥≤εg​,max(1,∣f(uk​)∣)∣f(uk+1​)−f(uk​)∣​≤εf​,∥uk+1​−uk​∥≤εu​,k≤kmax​.
(3)

Line Search and Armijo¶

Given a descent direction pkp_kpk​ with ∇f(uk)Tpk<0\nabla f(u_k)^T p_k<0∇f(uk​)Tpk​<0, line search picks αk>0\alpha_k>0αk​>0 to ensure robust decrease.

In general nonlinear problems, exact line search is usually too expensive; backtracking is the standard practical alternative.

Armijo backtracking¶

Choose α0>0\alpha_0>0α0​>0, c1∈(0,1)c_1\in(0,1)c1​∈(0,1), β∈(0,1)\beta\in(0,1)β∈(0,1). Set α←α0\alpha\leftarrow \alpha_0α←α0​ and reduce α←βα\alpha\leftarrow \beta\alphaα←βα until

f(uk+αpk)≤f(uk)+c1α∇f(uk)Tpk.f(u_k+\alpha p_k)\le f(u_k)+c_1\alpha \nabla f(u_k)^T p_k.f(uk​+αpk​)≤f(uk​)+c1​α∇f(uk​)Tpk​.
(10)

Interpretation:

  • right-hand side is a conservative linear prediction of decrease;

  • if condition fails, the step is too optimistic for local curvature;

  • backtracking is cheap and usually enough in practice.

Wolfe conditions (for 0<c1<c2<10<c_1<c_2<10<c1​<c2​<1):

f(uk+αpk)≤f(uk)+c1α∇f(uk)Tpk,∇f(uk+αpk)Tpk≥c2 ∇f(uk)Tpk.\begin{aligned} f(u_k+\alpha p_k) &\le f(u_k)+c_1\alpha \nabla f(u_k)^T p_k,\\ \nabla f(u_k+\alpha p_k)^T p_k &\ge c_2\,\nabla f(u_k)^T p_k. \end{aligned}f(uk​+αpk​)∇f(uk​+αpk​)Tpk​​≤f(uk​)+c1​α∇f(uk​)Tpk​,≥c2​∇f(uk​)Tpk​.​

Strong Wolfe replaces the curvature inequality with

∣∇f(uk+αpk)Tpk∣≤c2∣∇f(uk)Tpk∣.\left|\nabla f(u_k+\alpha p_k)^T p_k\right| \le c_2\left|\nabla f(u_k)^T p_k\right|.∣∣​∇f(uk​+αpk​)Tpk​∣∣​≤c2​∣∣​∇f(uk​)Tpk​∣∣​.

Geometric interpretation along the line ϕ(α):=f(uk+αpk)\phi(\alpha):=f(u_k+\alpha p_k)ϕ(α):=f(uk​+αpk​):

  • Armijo enforces sufficient decrease in function value (step not too large);

  • Wolfe curvature condition enforces ϕ′(α)\phi'(\alpha)ϕ′(α) to be much less negative than ϕ′(0)\phi'(0)ϕ′(0), so the step is not stopped too early;

  • strong Wolfe additionally controls oscillations by requiring small absolute slope at the accepted point.

Gradient Descent (GD)¶

Direction:

pk=−∇f(uk).p_k=-\nabla f(u_k).pk​=−∇f(uk​).
(13)

Strengths:

  • minimal per-iteration cost;

  • robust with Armijo backtracking;

  • simple baseline for any new model.

Limitations:

  • sensitive to conditioning;

  • slow on narrow valleys (zig-zag behavior);

  • linear convergence under strong convexity.

Use case in this course: reference solver for reduced OCP prototypes.

Nonlinear Conjugate Gradient (CG)¶

Replace steepest descent with

pk=−gk+βkpk−1,gk=∇f(uk),p_k=-g_k+\beta_k p_{k-1},\qquad g_k=\nabla f(u_k),pk​=−gk​+βk​pk−1​,gk​=∇f(uk​),
(14)

with reset pk=−gkp_k=-g_kpk​=−gk​ when needed.

Popular βk\beta_kβk​ formulas:

  • Fletcher-Reeves: βkFR=gkTgkgk−1Tgk−1\beta_k^{FR}=\frac{g_k^T g_k}{g_{k-1}^T g_{k-1}}βkFR​=gk−1T​gk−1​gkT​gk​​;

  • Polak-Ribiere+: βkPR+=max⁡ ⁣(0,gkT(gk−gk−1)gk−1Tgk−1)\beta_k^{PR+}=\max\!\left(0,\frac{g_k^T(g_k-g_{k-1})}{g_{k-1}^T g_{k-1}}\right)βkPR+​=max(0,gk−1T​gk−1​gkT​(gk​−gk−1​)​).

Key facts:

  • for SPD quadratics with exact line search, linear CG terminates in at most nnn steps;

  • nonlinear CG is memory-light (O(n) storage) and often faster than GD;

  • practical performance depends strongly on line-search quality and resets.

BFGS Quasi-Newton¶

BFGS stands for Broyden-Fletcher-Goldfarb-Shanno (the four authors who proposed this update family).

Core idea: mimic Newton’s method without computing the exact Hessian. Instead, build a sequence of positive-definite inverse-Hessian approximations from gradient differences, so directions keep curvature information while each iteration only needs function/gradient evaluations.

Approximate inverse Hessian Hk≈∇2f(uk)−1H_k\approx \nabla^2 f(u_k)^{-1}Hk​≈∇2f(uk​)−1 and use

pk=−Hkgk.p_k=-H_k g_k.pk​=−Hk​gk​.
(15)

With

sk=uk+1−uk,yk=gk+1−gk,ykTsk>0,s_k=u_{k+1}-u_k,\qquad y_k=g_{k+1}-g_k,\qquad y_k^T s_k>0,sk​=uk+1​−uk​,yk​=gk+1​−gk​,ykT​sk​>0,
(16)

BFGS update is

Hk+1=(I−ρkskykT)Hk(I−ρkykskT)+ρkskskT,ρk=1ykTsk.H_{k+1}=\left(I-\rho_k s_k y_k^T\right)H_k\left(I-\rho_k y_k s_k^T\right)+\rho_k s_k s_k^T, \quad \rho_k=\frac{1}{y_k^T s_k}.Hk+1​=(I−ρk​sk​ykT​)Hk​(I−ρk​yk​skT​)+ρk​sk​skT​,ρk​=ykT​sk​1​.
(17)

Equivalent expanded form:

Hk+1=Hk−HkykykTHkykTHkyk+skskTykTsk.H_{k+1} =H_k -\frac{H_k y_k y_k^T H_k}{y_k^T H_k y_k} +\frac{s_k s_k^T}{y_k^T s_k}.Hk+1​=Hk​−ykT​Hk​yk​Hk​yk​ykT​Hk​​+ykT​sk​sk​skT​​.
(18)

So the inverse-Hessian approximation is updated through rank-1 outer-product terms (overall a rank-2 correction per iteration).

Practical points:

  • much faster than GD on ill-conditioned smooth problems;

  • good local behavior (often superlinear with standard assumptions);

  • more memory and linear algebra than GD/CG.

For large scale: use L-BFGS (limited-memory variant): keep only the last mmm pairs (si,yi)(s_i,y_i)(si​,yi​) (typically m∈[5,20]m\in[5,20]m∈[5,20]), do not form HkH_kHk​ explicitly, and compute pk=−Hkgkp_k=-H_k g_kpk​=−Hk​gk​ via the two-loop recursion. Memory drops from O(n2)O(n^2)O(n2) to O(mn)O(mn)O(mn) and matrix factorizations are avoided.

Trust-Region (TR) Methods¶

Instead of fixing a direction then line-searching, solve a local model:

min⁡∥p∥≤Δkmk(p)=fk+gkTp+12pTBkp.\min_{\|p\|\le \Delta_k} m_k(p) =f_k+g_k^Tp+\frac12 p^T B_k p.∥p∥≤Δk​min​mk​(p)=fk​+gkT​p+21​pTBk​p.
(19)

Accept/reject with

ρk=f(uk)−f(uk+pk)mk(0)−mk(pk).\rho_k=\frac{f(u_k)-f(u_k+p_k)}{m_k(0)-m_k(p_k)}.ρk​=mk​(0)−mk​(pk​)f(uk​)−f(uk​+pk​)​.
(20)

Rule of thumb:

  • ρk\rho_kρk​ small: model inaccurate -> shrink radius Δk\Delta_kΔk​;

  • ρk\rho_kρk​ good: accept step, maybe enlarge Δk\Delta_kΔk​.

Advantages:

  • very robust under nonconvexity or poor Hessian quality;

  • natural handling of negative curvature.

Typical subproblem solvers: Cauchy step, dogleg, truncated CG.

Method Selection Cheat Sheet¶

  • GD + Armijo: safest baseline, cheapest iteration, slowest asymptotically.

  • Nonlinear CG: cheap memory, often better than GD, more tuning-sensitive.

  • BFGS: strong default for smooth medium-size problems.

  • Trust-region: robust when curvature is difficult/nonconvex.

In reduced PDE-constrained optimization:

  • dominant cost = gradient/Hessian-vector evaluation (state/adjoint solves);

  • iteration count matters less than “PDE solves per accepted step”;

  • robust globalisation is critical.
